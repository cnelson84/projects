Introduction

In this project, Ill demonstrate my skills and techniques to construct my own XGBoost classification model. The XGBoost model is a very powerful extension of decision trees, so having a strong working familiarity with this process will strengthen my skills and resume as a data professional. This project is a continuation of the airlines project in which I built decision tree and random forest models.

  • The Purpose: To train, tune, and evaluate an XGBoost model.
  • The Goal: To compare the performance of all three models and decide which model is best. I'll also explore the feature importances of this model and identify the features that most contribute to customer satisfaction.

Step 2: Model Preparation

Before I proceed with modeling, I'll to consider which metrics I will ultimately want to leverage to evaluate this model.

  • As this is a binary classfication problem, it will be important to evaluate not just accuracy, but the balance of false positives and false negatives that the model's predictions provide. Therefore, precision, recall, and ultimately the F1 score will be excellent metrics to use.
  • The ROC AUC (Area Under the Receiver Operating Characteristic) score is also suited to this type of modeling.

Preparing the data for predictions

  • I noticed when previewing the data that there are several non-numerical variables (object data types) within this dataset.
  • To prepare this DataFrame for modeling, I'll first convert these variables into a numerical format.
In[6]:
# Converting the object predictor variables to numerical dummies.
airline_data_dummies = pd.get_dummies(airline_data, 
                                         columns=['satisfaction','Customer Type','Type of Travel','Class'])

Isolating the target and predictor variables

In[7]:
# Defining the y (target) variable.
y = airline_data_dummies['satisfaction_satisfied']

# Defining the X (predictor) variables.
X = airline_data_dummies.drop(['satisfaction_satisfied','satisfaction_dissatisfied'], axis = 1)

Dividing the data

  • I'll divide the data into a training set (75% of the data) and test set (25% of the data). This is an important step in the process, as it allows me to reserve a part of the data that the model has not used to test how well the model generalizes (or performs) on new data.
In[8]:
# Performing the split operation on the data.
# Assigning the outputs as follows: X_train, X_test, y_train, y_test.
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size = 0.25, random_state = 0)

Step 3: Model Building

"Instantiating" the XGBClassifer

  • Before I fit the model to the airline dataset, I'll first create the XGB Classifier model and define its objective. I'll use this model to fit and score different hyperparameters during the GridSearch cross-validation process.
In[9]:
# Defining xgb to be the XGBClassifier.
xgb = XGBClassifier(objective='binary:logistic', random_state=0)

Defining the parameters for hyperparameter tuning

  • To identify suitable parameters for the xgboost model, I'll first define the parameters for hyperparameter tuning. Specifically considering max_depth, min_child_weight, learning_rate, n_estimators, subsample, and/or colsample_bytree.
In[12]:
# Defining parameters for tuning as `cv_params`.

               'n_estimators': [5,10,15],

Observations:

  • More estimators will initially improve the model's performance. However, increasing the number of estimators will also considerably increase the time spent during the GridSearch process, and there will be diminishing returns as the number of estimators continues to increase.

Defining how the models will be evaluated

  • Here I'll define how the models will be evaluated for hyperparameter tuning. To yield the best understanding of model performance, I'll utilize a suite of metrics.
In[13]:
# Defining my criteria as `scoring`.
scoring = {'accuracy', 'precision', 'recall', 'f1'}

Constructing the GridSearch cross-validation

In[14]:
# Constructing GridSearch.
xgb_cv = GridSearchCV(xgb,
                      cv_params,
                      scoring = scoring,
                      cv = 5,
                      refit = 'f1'
                     )

Step 4: Results and Evaluation

Formulating predictions on the test set

  • To evaluate the predictions yielded from the model, I'll leverage a series of metrics and evaluation techniques from scikit-learn by examining the actual observed values in the test set relative to the model's prediction.
In[17]:
# Applying the model to predict on the test data.
y_pred = xgb_cv.predict(X_test)

Leveraging metrics to evaluate the model's performance

  • I'll apply a series of metrics from scikit-learn to assess the model. Specifically, by printing the accuracy score, precision score, recall score, and f1 score associated with the test data and predicted values.
In[18]:
# 1. Print the accuracy score.
ac_score = metrics.accuracy_score(y_test, y_pred)
print('accuracy score:', ac_score)

# 2. Print the precision score.
pc_score = metrics.precision_score(y_test, y_pred)
print('precision score:', pc_score)

# 3. Print the recall score.
rc_score = metrics.recall_score(y_test, y_pred)
print('recall score:', rc_score)

# 4. Print the f1 score.
f1_score = metrics.f1_score(y_test, y_pred)
print('f1 score:', f1_score)

accuracy score: 0.9346165691407453

recall score: 0.9331092436974789

Observations:

  • The accuracy score for this model is 0.934 or 93.4% accurate.
  • In classification problems, accuracy is useful to know but may not be the best metric to evaluate this model.
  • Precision and recall scores are both useful to evaluate the correct predictive capability of the model because they balance the false positives and false negatives inherent in prediction. The model shows a precision score of 0.947, suggesting the model is very good at predicting true positives. This means the model correctly predicts whether the airline passenger will be satisfied. The recall score of 0.933 is also very good. This means that the model does a good job of correctly identifying dissatisfied passengers within the dataset. These two metrics combined give a better assessment of model performance than the accuracy metric does alone.
  • The F1 score balances the precision and recall performance to give a combined assessment of how well this model delivers predictions. In this case, the F1 score is 0.940, which suggests very strong predictive power in this model.

Gaining clarity with the confusion matrix

  • A confusion matrix is a graphic that shows a model's true and false positives and true and false negatives. It helps to create a visual representation of the components feeding into the metrics above.
In[19]:
# Constructing the confusion matrix for the predicted and test values.

# Creating the display for the confusion matrix.
disp = metrics.ConfusionMatrixDisplay(confusion_matrix=cm, display_labels=xgb_cv.classes_)

# Plotting the visual in-line.
disp.plot()

  • Additionally, the concentration of true positives and true negatives stands out relative to false positives and false negatives, respectively. This ratio is why the precision score is so high (0.947).

Considerations

Takeaways:

  • The evaluation of the model is important to inform if the model has delivered accurate predictions.
  • Splitting the data is important for ensuring that there is new data for the model to test its predictive performance.
  • Each metric provides an evaluation from a different standpoint, and accuracy alone is not a strong way to evaluate a model.
  • Effective assessments balance the true/false positives versus true/false negatives through the confusion matrix and F1 score.

Steps to take in sharing findings with others:

  • Showcase the data used to create the prediction and the performance of the model overall.
  • Review the sample output of the features and the confusion matrix to reference the model's performance.
  • Highlight the metric values, emphasizing the F1 score.
  • Visualize the feature importance to showcase what drove the model's predictions.

Recommendations to stakeholders:

  • The model created is highly effective at predicting passenger satisfaction.
  • The feature importance of seat comfort warrants additional investigation. It will be important to ask domain experts why they believe this feature scores so highly in this model.
