Step 9: Advanced Joins for Customer Segmentation¶

I'll classify customers into New, Regular, and VIP¶

  • Splits customers into 3 equal percentile groups (New, Regular, VIP)
  • Used JOIN on OrderCount to assign percentile ranks
  • Ensures even distribution of customers
  • Used LEFT JOIN to include all customers
  • Ensures even those without purchases are assigned a category
  • Ranked customers within each category by TotalSpent
  • Ensures 10 customers per category based on spending
In [32]:
query = """
WITH CustomerOrders AS (
    SELECT 
        c.CustomerId,
        c.FirstName || ' ' || c.LastName AS Customer,
        COUNT(i.InvoiceId) AS OrderCount,
        SUM(i.Total) AS TotalSpent
    FROM Customers c
    LEFT JOIN Invoices i ON c.CustomerId = i.CustomerId
    GROUP BY c.CustomerId
),
PercentileRanks AS (
    SELECT 
        OrderCount,
        NTILE(3) OVER (ORDER BY OrderCount) AS PercentileGroup
    FROM CustomerOrders
),
CategorizedCustomers AS (
    SELECT 
        c.CustomerId,
        c.Customer,
        c.OrderCount,
        c.TotalSpent,
        CASE 
            WHEN p.PercentileGroup = 1 THEN 'New'
            WHEN p.PercentileGroup = 2 THEN 'Regular'
            ELSE 'VIP'
        END AS CustomerCategory
    FROM CustomerOrders c
    JOIN PercentileRanks p ON c.OrderCount = p.OrderCount
),
RankedCustomers AS (
    SELECT 
        Customer, OrderCount, TotalSpent, CustomerCategory,
        ROW_NUMBER() OVER (PARTITION BY CustomerCategory ORDER BY TotalSpent DESC) AS Rank
    FROM CategorizedCustomers
)
SELECT Customer, OrderCount, TotalSpent, CustomerCategory
FROM RankedCustomers
WHERE Rank <= 10
ORDER BY CustomerCategory, TotalSpent DESC;
"""

customer_segments = pd.read_sql(query, conn)
print(customer_segments)
